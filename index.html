Let's look at one of the issues here...
Anchor as button, no href attribute
Probably has JavaScript attached, on click
No href attribute, no focus, no keyboard access
Also affects screenreader users
Make it focusable:
Add href attribute to anchor
Or tabindex attribute to any element
Better: Use button element, anchor only for actual links
Enhance with JavaScript
Two more examples of bad markup...

Markup for links and buttons
Product title should be a header.
Product image needs an alt attribute with a useful description.
The decoration image needs an alt attribute with an empty value. That tells screenreaders to ignore this image.
If there is no description, use empty alt attribute. Otherwise: Screenreader reads the image filename. Sounds terrible.
One more example...

Markup for content
Found on another airline site
Link labels to inputs using the for-attribute.
Screenreader can then announce the label on focus of the input
"search edit text" instead of "edit text"
Also increases hit are for mouse users
These are trivial, but often missed
